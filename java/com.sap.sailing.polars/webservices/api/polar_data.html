SAP Sailing Analytics Webservices API Version 1.0

URL: /api/v1/regattas/{regattaname}/races/{racename}/competitordata

Description:

Gets the binary representation of the server's polar data. By default, data for all boat classes is returned. Optionally, the results can be filtered to specific boat classes using the boatClassName query parameter, which may be specified multiple times to include several boat classes.

You need the POLAR_DATA:READ permission for this. The output can be deserialized into a Java object of type com.sap.sailing.polars.impl.PolarDataServiceImpl using its initiallyFillFromInternal method. It is the same format used by the cross-server replication of the polar data. See also the com.sap.sailing.polars.jaxrs.client.PolarDataClient class.


Webservice Type: REST
Webservice Method: GET
Output format: Binary Java Object
Query Parameters:
NameRequiredDescription
boatClassName No Name of a boat class to include in the response. May be specified multiple times to request data for several boat classes. If omitted, data for all boat classes is returned.
Examples: All boat classes:
curl -o polar.dat -H 'Authorization: Bearer {your-access-token-here}' https://www.sapsailing.com/polars/api/polar_data

Filtered to specific boat classes:
curl -o polar.dat -H 'Authorization: Bearer {your-access-token-here}' "https://www.sapsailing.com/polars/api/polar_data?boatClassName=49er&boatClassName=Nacra17"
